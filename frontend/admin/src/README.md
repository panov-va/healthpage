# Admin — Feature-Sliced Design

Слои (сверху вниз, импорт только «вниз»):

- `app/` — инициализация приложения (точка входа, провайдеры, роутер).
- `processes/` — сквозные пользовательские процессы (напр. онбординг).
- `pages/` — страницы-композиции.
- `widgets/` — самостоятельные блоки UI.
- `features/` — действия пользователя (создать страницу, сменить статус компонента).
- `entities/` — бизнес-сущности (component, componentGroup, incident, ...).
- `shared/` — переиспользуемое без бизнес-логики (ui-kit, api-клиент, утилиты).

Типы API импортируются из `shared/api-types` (генерируются из `openapi.yaml`,
руками не правятся). FSD соблюдать строго — см. CLAUDE.md §3, §7.
